# Activities And Feed API

Run the common setup in [README](./README.md), then login with [Auth](./users-auth.md) to set `ACCESS_TOKEN`.

## Setup

Activities are generated by write operations in other modules. Use existing IDs that match your local data.

```bash
SUBJECT_ID="000212ed-b7b4-45c1-8d0d-7cd72906baa4"
TARGET_USER_ID=2
PUBLIC_USER_ID=2
```

Get your own user ID:

```bash
MY_USER_ID=$(
  curl -s -X GET "$BASE_URL/api/users/me/profile/" \
    -H "Authorization: Bearer $ACCESS_TOKEN" \
    | tee /tmp/noshiro_my_profile.json | jq -r ".data.user_id // empty"
)

cat /tmp/noshiro_my_profile.json | jq
echo "$MY_USER_ID"
```

Supported activity types:

```text
user_subject_created
user_subject_updated
review_created
collection_created
collection_item_added
user_followed
```

## Generate Activities

Create or update a marked subject:

```bash
curl -s -X POST "$BASE_URL/api/users/me/subjects/" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"subject_id\": \"$SUBJECT_ID\",
    \"status\": \"doing\",
    \"is_public\": true
  }" | jq
```

Create a review:

```bash
curl -s -X POST "$BASE_URL/api/users/me/subjects/$SUBJECT_ID/reviews/" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Activity review",
    "content": "This review should create an activity.",
    "is_public": true,
    "is_spoiler": false
  }' | jq
```

Create a collection:

```bash
ACTIVITY_COLLECTION_ID=$(
  curl -s -X POST "$BASE_URL/api/users/me/collections/" \
    -H "Authorization: Bearer $ACCESS_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Activity collection",
      "is_public": true
    }' | tee /tmp/noshiro_activity_collection.json | jq -r ".data.id // empty"
)

cat /tmp/noshiro_activity_collection.json | jq
echo "$ACTIVITY_COLLECTION_ID"
```

Add a collection item:

```bash
curl -s -X POST "$BASE_URL/api/users/me/collections/$ACTIVITY_COLLECTION_ID/items/" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"subject_id\": \"$SUBJECT_ID\",
    \"order\": 1,
    \"relation\": \"activity test\"
  }" | jq
```

Follow a user:

```bash
curl -s -X POST "$BASE_URL/api/users/me/following/$TARGET_USER_ID/" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

## List My Activities

This endpoint is paginated and includes your own public and private activities.

```bash
curl -s -X GET "$BASE_URL/api/users/me/activities/?page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Filter by activity type:

```bash
curl -s -X GET "$BASE_URL/api/users/me/activities/?activity_type=review_created&page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Ordering:

```bash
curl -s -X GET "$BASE_URL/api/users/me/activities/?ordering=-created_at&page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Supported ordering values:

```text
id
-id
created_at
-created_at
```

## Public User Activities

This endpoint is paginated and returns only currently public activities.

```bash
curl -s -X GET "$BASE_URL/api/users/$PUBLIC_USER_ID/activities/?page=1&page_size=16" | jq
```

Filter by activity type:

```bash
curl -s -X GET "$BASE_URL/api/users/$PUBLIC_USER_ID/activities/?activity_type=collection_created&page=1&page_size=16" | jq
```

Missing user, expected to fail:

```bash
curl -s -X GET "$BASE_URL/api/users/999999/activities/" | jq
```

Expected business error:

```json
{
  "code": 11201,
  "message": "user not found",
  "data": null
}
```

## My Feed

This endpoint is paginated and returns public activities from users you follow.

```bash
curl -s -X GET "$BASE_URL/api/users/me/feed/?page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Include your own public activities:

```bash
curl -s -X GET "$BASE_URL/api/users/me/feed/?include_self=true&page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Filter by activity type:

```bash
curl -s -X GET "$BASE_URL/api/users/me/feed/?activity_type=user_followed&include_self=true&page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

## Privacy Checks

Private subject activity should appear in my activities but not public activities or feed:

```bash
curl -s -X POST "$BASE_URL/api/users/me/subjects/" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"subject_id\": \"$SUBJECT_ID\",
    \"status\": \"doing\",
    \"is_public\": false
  }" | jq
```

Check my activities:

```bash
curl -s -X GET "$BASE_URL/api/users/me/activities/?activity_type=user_subject_updated&page=1&page_size=16" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq
```

Check public activities:

```bash
curl -s -X GET "$BASE_URL/api/users/$MY_USER_ID/activities/?activity_type=user_subject_updated&page=1&page_size=16" | jq
```

Expected result for public activities: private subject updates are not included.

Private collection activity should not appear in public activities:

```bash
PRIVATE_ACTIVITY_COLLECTION_ID=$(
  curl -s -X POST "$BASE_URL/api/users/me/collections/" \
    -H "Authorization: Bearer $ACCESS_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Private activity collection",
      "is_public": false
    }' | tee /tmp/noshiro_private_activity_collection.json | jq -r ".data.id // empty"
)

cat /tmp/noshiro_private_activity_collection.json | jq
echo "$PRIVATE_ACTIVITY_COLLECTION_ID"

curl -s -X GET "$BASE_URL/api/users/$MY_USER_ID/activities/?activity_type=collection_created&page=1&page_size=16" | jq
```

Expected result: `Private activity collection` is not included in public activities.

## Frontend Flow

On the current user's activity page:

1. Load `GET /api/users/me/activities/`.
2. Render activity cards by `activity_type`.
3. Use nested `subject`, `review`, `collection`, and `target_user` objects for links.

On a public user page:

1. Load `GET /api/users/{user_id}/activities/` only when the activity tab is opened.
2. Treat this list as public data; private activity is filtered by the backend.

On the home feed:

1. Load `GET /api/users/me/feed/`.
2. Use `include_self=true` if the product wants the viewer's own public actions mixed into the feed.
